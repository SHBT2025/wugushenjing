# 无股神经 (stockrsi3.1)

> A股智能分析工具 — RSI 精准填充，日周月独立分析

---

## 1. 项目目标

### 核心目标

开发一个面向 A 股个人投资者的轻量级 **K 线 + RSI 技术分析工具**，帮助用户快速判断股票的超买超卖状态，辅助交易决策。

### 关键指标

| 指标 | 说明 |
|------|------|
| **RSI(6)** | 6 周期相对强弱指数，Wilder 平滑算法，识别超买(>80) / 超卖(<20) |
| **日/周/月** | 三个时间维度独立分析，互不干扰 |
| **电池条** | 10 格可视化 RSI 强度，红涨绿跌，50 为中轴 |
| **事件标记** | 在 K 线图上标记买入/卖出/自定义事件，关联历史 RSI 值 |

### 数据源

- **实时行情 & K 线数据**：东方财富公开 API（push2.eastmoney.com）
- **股票搜索**：东方财富 suggest 接口（searchapi.eastmoney.com）
- **存储**：浏览器 localStorage（3 个独立 key）

### 用户群体

A 股个人投资者，偏好 RSI 指标进行短线/中线交易决策，需要一个干净、无干扰、可离线使用的看盘工具。

## 4. 功能模块

### 4.1 自选股管理

| 功能 | 实现 |
|------|------|
| 添加自选股 | 输入代码/拼音/名称搜索，下拉选择 |
| 删除自选股 | 列表项 hover 显示 × 按钮 |
| 排序 | 拖拽排序（SortableJS） |
| 选中切换 | 点击列表项切换当前查看的股票 |
| 持久化 | `localStorage.watchlist_mid2` |

### 4.2 股票搜索

| 功能 | 实现 |
|------|------|
| 搜索方式 | 股票代码 / 拼音首字母 / 中文名称 |
| API | 东方财富 suggest 接口 |
| 渲染 | 下拉弹窗，显示代码 + 名称 + 市场标记 |
| 防抖 | 300ms 输入防抖 |

### 4.3 K 线图（核心）

| 功能 | 实现 |
|------|------|
| 周期切换 | 日线 / 周线 / 月线（独立数据请求） |
| K 线绘制 | Canvas 原生绘制，含价格坐标轴 |
| 十字光标 | 鼠标悬停显示十字线 + 横向 Tooltip |
| 加载状态 | 加载时显示遮罩 + 旋转动画 |
| 历史数据 | 最多 500 条 |
| 成交量 | K 线底部成交量柱（按比例缩放） |

### 4.4 RSI 指标（核心）

| 功能 | 实现 |
|------|------|
| 算法 | Wilder's Smoothing RSI(6) |
| 子图 | K 线下方独立 RSI 区域，60% / 20% 分界 |
| 填充 | RSI 曲线下方区域，超买红色 / 超卖绿色 |
| 电池条 | 10 格可视化，50 为中轴，红涨绿跌 |
| 数值显示 | 光标移动时 Tooltip 显示当前 RSI 值 |

### 4.5 事件系统

| 功能 | 实现 |
|------|------|
| 事件类型 | 买入 (buy) / 卖出 (sell) / 标记 (mark) |
| 事件创建 | 弹窗表单，填写日期 + 类型 + 备注 |
| 事件表格 | 可排序表格，显示日期、类型、价格、RSI |
| K 线标记 | 对应日期 K 线上方显示标记（B/S/M） |
| 事件值 | 自动计算事件日期当天的收盘价 + RSI |
| 周期绑定 | 事件绑定到特定周期（日/周/月） |
| 跨股票 | 同一事件列遍历所有自选股取值 |

### 4.6 提醒系统

| 功能 | 实现 |
|------|------|
| 价格提醒 | 高于某价 / 低于某价（基于实时行情） |
| RSI 提醒 | 日 RSI / 周 RSI / 月 RSI 阈值触发 |
| 提醒冷却 | 同类型提醒 1 小时内不重复 |
| 定时检查 | 工作日 10:32 / 14:32 / 15:57 |
| 通知弹窗 | 自定义 Toast 样式弹窗 |

### 4.7 实时行情

| 功能 | 实现 |
|------|------|
| 刷新方式 | 定时轮询（约 3 秒） |
| 数据内容 | 最新价 / 涨跌幅 / 涨跌额 |
| 显示位置 | 工具栏区域 + 事件表格实时列 |
| 颜色标记 | 涨红色 / 跌绿色 |
| 缓存 | `liveDataCache` 减少重复请求 |

### 4.8 数据持久化

| 存储项 | Key | 内容 |
|--------|-----|------|
| 自选股列表 | `watchlist_mid2` | `[{code, name}]` |
| 事件数据 | `eventSystem_mid2` | `{globalEvents, stockEventValues}` |
| 提醒设置 | `stockAlerts_mid2` | `[[code, settings], ...]` |

### 4.9 导入导出

| 功能 | 实现 |
|------|------|
| 导出 | 下载 JSON 文件（含全部 3 个 localStorage key） |
| 导入 | 上传 JSON 文件并覆盖本地数据 |

### 4.10 PWA 支持

| 功能 | 实现 |
|------|------|
| 离线缓存 | Service Worker 缓存优先策略 |
| 可安装 | manifest.json，独立窗口模式 |
| 更新提示 | 检测 SW 更新，刷新后激活 |

## 6. 禁止事项

### ❌ 架构禁止

1. **禁止拆分 index.html** — 本项目是单文件应用，所有 HTML/CSS/JS 必须保持内联。不得拆分为单独的 .css 或 .js 文件。
2. **禁止引入构建工具** — 不得使用 Webpack / Vite / Gulp 等构建工具。保持零构建、零依赖。
3. **禁止引入新外部库** — 除 SortableJS（已在 CDN 加载）外，不得新增任何外部 JS/CSS 库（包括 FontAwesome、Chart.js、jQuery、Bootstrap 等）。
4. **禁止后端服务** — 不得添加 Node.js / Python / PHP 等后端。应用必须是纯静态文件。
5. **禁止修改数据源架构** — 不得将数据源从东方财富切换为需要 API Key / 注册的服务。保持免费公开接口。

### ❌ 代码禁止

6. **禁止使用 TypeScript** — 必须使用原生 JavaScript。
7. **禁止使用 ES Modules** — 不得使用 `import` / `export` 语法。所有代码必须在全局作用域内。
8. **禁止操作项目文件结构** — 不得在 `stockrsi3.1/` 目录外创建或修改任何文件。所有更改仅限于该目录。
9. **禁止修改 service-worker.js 的缓存策略** — SW 的缓存优先策略（Cache First）不得改为网络优先或 Network Only。

### ❌ 功能禁止

10. **禁止添加登录/注册** — 本项目不需要用户认证系统。
11. **禁止添加付费/广告** — 项目中不得出现任何付费功能或广告位。
12. **禁止添加实时推送** — 不得引入 WebSocket 或 SSE，保持 HTTP 轮询模式。
13. **禁止修改 RSI 算法** — RSI(6) Wilder's Smoothing 算法不得改为其它算法（如 EMA、SMA 变体）。
14. **禁止删除 localStorage 持久化机制** — 现有 3 个 localStorage key 的结构不得合并或删除。
15. **禁止去除 PWA 功能** — Service Worker 和 manifest.json 必须保留，不可移除。
